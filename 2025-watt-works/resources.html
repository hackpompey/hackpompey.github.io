The Challenges

Choose a challenge to work on, designed to transform how people understand and interact with the electricity network

Challenge 1
Watt’s Happening?

Transform invisible network problems into compelling visual experiences

Design an engaging visualization that reveals voltage stability issues OR network constraints to non-technical audiences. Your solution should make people care about a problem they can't see.

Focus on ONE specific aspect (voltage fluctuations, peak demand, renewable integration) and make it crystal clear through interactive dashboards, data animations, or creative visual metaphors.

Background

As the distribution network evolves, the rapid increase in domestic solar PV, electrical vehicles, heat pumps, and home batteries is creating new voltage stability challenges. These fluctuations are often invisible to the public and difficult to quantify for many stakeholders. Without clear visualisation, it is hard to build shared understanding, influence policy, or inspire collaborative action.

Starter Project Idea

Make a visualisation that shows the impact of voltage fluctuations. Use a heartbeat as a visual metaphor to demonstrate the grid’s health. When voltage is stable, it "breathes" steadily. When unstable, it shows visible "stress".

Challenge 2
Flex Quest

Make demand flexibility fun and understandable through play

Create a game or interactive experience that teaches how grid flexibility works. Players should understand concepts like peak shifting, load balancing, or flexible assets through gameplay - not lectures. Think mobile game mechanics, physical card games, or interactive simulations that work for audiences from age 10 to 60.

Background

Flexibility is a cornerstone of a smarter, greener energy system. However, the concept remains abstract and inaccessible to many customers. As Distribution Network Operator’s (DNO’s) seek to engage more people in flexibility services, there’s a need to make these ideas tangible, fun, and easy to understand.

Starter Project Idea

Make a card game where players are grid operators trying to keep supply and demand balanced while dealing with unpredictable renewable energy and demand spikes like wind surges, solar dips, factory shifts and heat waves.

Give players tools (flexibility services) to maintain grid stability, like battery storage, the ability to pause non-essential loads, or to delay/accelerate EV charging.

Reward those that keep the grid balanced, penalize instability.

Near Real-time Data Access (NeRDA)

NeRDA provides live electricity demand and network health measurements from substations across high and low voltage networks (updated every 3-10 minutes). You can use this for getting grid conditions in near real-time to identify flexibility opportunities, forecast constraints, and build applications that respond to actual network supply and demand.

There are two parts. The Portal (or Dashboard) which provides a GUI over the API, and then the API itself. We recommend using the GUI to find areas and substations of interest, then taking the substation id from the URL to query the API as per the API guides linked below.

Hack Pompey’s Technical Notes on the API

Request authentication

We recommend authenticating API requests using the "Short-Term API Key" by including the { Authorization: Bearer YOUR_API_KEY } request header.

The "Long-Term API Key" requires making GET requests with a body which is not supported by most request libraries or apis, including the native fetch api in JavaScript.

Just keep in mind that the short-term api key will expire after a session, so it is wise to grab a new short-term api key before demoing your app for example.

CORS issues

If you are calling the NeRDA directly from the frontend of your app or website, you will likely encounter CORS errors. (CORS explained in 100 seconds on Youtube.)

Calls to NeRDA are okay from a server however. To save you the trouble of setting up a server if you don't need one, we’ve created a proxy server to forward your requests for you so that you can avoid CORS errors and call NeRDA from the frontend.

It’s a drop in replacement, you just need to swap out the domain segment of the API URL as follows:

Before:
https://nerda-prod-apis-v2.azurewebsites.net/api/ApiNerdaStatic?substation=74f42299-9f8e-4cb4-922c-0e3273bff4c7

After:
https://nerda.hackathons.dev/api/ApiNerdaStatic?substation=74f42299-9f8e-4cb4-922c-0e3273bff4c7

Smart Meter Data

Electricity consumption data from neighbourhoods (aggregated from 5+ properties), which would enable you to analyze real household energy usage patterns, build forecasting models, and understand grid demand at the local network level.

Dataset and further information

Embedded Capacity Register (ECR)

The Embedded Capacity Register shows all generation and storage assets (≥50kW) connected or waiting to connect to the distribution network, plus their flexibility services, helping you understand grid capacity and supply, identify network constraints, and spot opportunities for energy trading or new connections.

Dataset and further information

SSEN Substation Data

The Substations Dataset provides the location, type, and identification of all SSEN substations (pole and ground-mounted) across Scotland and Southern England, cand be used to map the physical network infrastructure, analyze spatial distribution of supply points, and plan renewable energy connections or emergency response strategies.
